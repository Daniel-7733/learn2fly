# Learn2Fly Roadmap

## Rule of the Project

Do not rush to hardware.

The project grows in this order:

1. Simulate
2. Stabilize
3. Log data
4. Analyze
5. Add C/C++
6. Add learning
7. Add hardware

---

## Phase 0 — Setup

Goal: Create the project foundation.

Tasks:

- Create GitHub repository
- Add README.md
- Add .gitignore
- Create basic folders
- Make first commit
- Write the first project journal entry

Success condition:

- The repository exists on GitHub
- The project has a clear purpose
- You can push changes safely

---

## Phase 1 — Falling Plane

Goal: Simulate gravity.

Tasks:

- Create Plane class
- Add altitude, speed, angle
- Create update loop
- Reduce altitude over time
- Stop when altitude reaches 0

Success condition:

- The program prints altitude decreasing over time

---

## Phase 2 — Basic Flight State

Goal: Add simple movement.

Tasks:

- Add x and y position
- Add velocity
- Add throttle
- Add simple lift
- Add crash condition

Success condition:

- The plane can either stay in air briefly or crash depending on values

---

## Phase 3 — Data Logging

Goal: Record what happens during flight.

Tasks:

- Save time, altitude, speed, angle, throttle to CSV
- Create a flight_log.csv file
- Review the data after each run

Success condition:

- Every simulation creates readable flight data

---

## Phase 4 — Rule-Based Autopilot

Goal: Build a simple non-AI pilot.

Tasks:

- Choose target altitude
- Increase throttle when altitude is too low
- Decrease throttle when altitude is too high
- Try to keep the plane stable

Success condition:

- The plane survives longer than manual/default simulation

---

## Phase 5 — C or C++ Control Logic

Goal: Move important control logic to a faster systems language.

Tasks:

- Write small C/C++ function for control decision
- Compile it separately
- Connect it later to Python
- Keep it small and understandable

Success condition:

- Python can use control values generated by C/C++

---

## Phase 6 — Learning Pilot

Goal: Python studies flight data.

Tasks:

- Read previous flight logs
- Compare successful and failed flights
- Detect patterns
- Suggest better throttle/angle decisions

Success condition:

- Python can explain why some flights failed

---

## Phase 7 — Hardware Research

Goal: Prepare for real-world flying.

Tasks:

- Learn basic electronics
- Learn about IMU sensors
- Learn about servos and motors
- Study microcontrollers
- Study safe testing

Success condition:

- You understand what hardware is needed before buying anything

---

## First 7-Day Plan

Day 1:
Create GitHub repo and push starter files.

Day 2:
Create `Plane` class.

Day 3:
Simulate gravity and falling.

Day 4:
Add speed and angle.

Day 5:
Add simple update loop.

Day 6:
Add CSV logging.

Day 7:
Write what you learned in `notes/dev_journal.md`.
